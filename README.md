  ШАГ 1 — собрать ветки в dev:            ШАГ 2 — влить dev в main:
  ──────────────────────────────          ──────────────────────────────
  git checkout dev                        git checkout main
  git pull origin dev                     git pull origin main
  git merge feature_ivan                  git merge dev
  git merge feature_<участник1>           git commit -m "release(main): день N"
  git merge feature_<участник2>           git push origin main
  git commit -m "merge(dev): день N"
  git push origin dev
  (проверить сайт в браузере!)

  Только после ШАГ 2 участники делают:
  git pull origin main     ← получить актуальную версию от всей команды

  ИТОГ: 3 УРОВНЯ ЗАЩИТЫ MAIN
══════════════════════════════════════════════════════════════════════════════════════════════════════

  feature_*  →  никто не пишет код напрямую в main или dev
     └──►  dev  →  командир собирает и проверяет перед отправкой
              └──►  main  →  только стабильный, проверенный код

  feature_*  =  рабочая зона            (каждый в своей ветке)
  dev        =  буферная зона           (командир собирает и тестирует)
  main       =  стабильная зона         (только финальные версии)
